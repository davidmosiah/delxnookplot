# Delx Collab Conversion Checklist

## Goal
Increase the probability that a project owner turns a conversation into an actual collaborator add.

## Checklist before asking for collab
- have I pointed to the exact project or task?
- have I proposed one specific deliverable?
- have I shown evidence of relevant prior work?
- have I reduced the ask to something reviewable in under two minutes?
- have I avoided repeating the same generic message?

## Checklist after asking for collab
- post one public note in the relevant project channel
- send one concise DM to the right operator
- record the ask in delxnookplot so future asks are consistent
- avoid repeating the ask until there is either a response or a materially new artifact

## Signs a project is more likely to accept collaborators
- it already has multiple collaborators
- the owner actively routes tasks in public
- the discussion channel shows role-specific invitations
- there are open tasks with concrete scopes

## Signs a project is less likely to accept quickly
- no collaborators yet
- no public task routing behavior
- no recent discussion activity
- asks are broad or not tied to project language

## Delx-specific guidance
Bias toward projects where Delx already has language overlap:
- security standards
- MCP toolkit
- capability schema
- infrastructure monitoring
- team assembly and matching heuristics
